Agentic RAG

一个支持多格式文档解析、向量检索、证据展示与 Agent 多轮检索问答的课程项目。

GitHub 本地运行

项目目标

面向课程场景实现一个可扩展的 Agentic RAG 系统，支持知识库构建、检索增强问答、证据展示，以及多轮问题分解与回答生成。

在线部署说明

当前网页通过 GitHub Actions 自动部署到 GitHub Pages，用于展示项目介绍、架构和使用方式。 由于 GitHub Pages 仅支持静态页面，仓库中的 Streamlit 应用需要部署到其他支持 Python 的平台。

核心技术

LangGraph、LangChain、Chroma、Sentence Transformers、Streamlit、OpenAI Compatible API。

Features

主要能力

多格式文档接入

支持 txt、md、pdf、doc/docx、xlsx、ppt/pptx 等常见文档格式。

知识库构建

支持文档解析、切分、Embedding 生成与向量库写入。

Agent 多轮检索

能够执行问题分析、子问题拆解、多轮检索与结果整合。

证据可视化

展示召回片段、引用来源和执行状态，便于评估回答质量。

Architecture

项目结构

app/
├─ agent/       # Agent 图、节点、状态、提示词
├─ config/      # 配置与日志
├─ loaders/     # 多格式文档加载器
├─ rag/         # Embedding、切分、检索、向量库
├─ services/    # 知识库构建与问答服务
└─ ui/          # Streamlit 界面

data/
└─ knowledge_base/
   ├─ raw/      # 本地知识库原始文档（默认不提交）
   └─ processed/

site/
└─ GitHub Pages 项目主页
Run

本地运行

python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
streamlit run app/ui/streamlit_app.py

运行前请在本地配置 `.env`，并准备好知识库文档与所需 API Key。`.env`、原始文档、模型与向量库均建议保留在本地，不提交到 GitHub。
